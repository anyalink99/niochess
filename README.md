# niochess

Real-time chess with no turns. Pieces don't teleport — they **fly** from one square to another over a fixed time (~10 seconds by default, independent of distance). While a piece is in flight it is invulnerable and doesn't capture. The capture happens **on arrival**, when the piece lands. Whoever captures the king first wins.

Play it: **https://anyalink99.github.io/niochess/**

## How to play

- Click a piece, then a highlighted square — the piece flies there. You can also drag it.
- In flight a piece is invulnerable and occupies no square.
- On arrival: lands on an enemy → capture; two enemies on one square → both die; running into your own occupied square → the arriving piece is lost; a pawn reaching the far rank → queen.
- No check or mate — the goal is to capture the king.
- The AI doesn't start moving until you press **New game**.

## Modes

- **Single** — you (White) against the AI (Black). AI behaviour is tunable via sliders: aggression, sniping (interception), king reaction time, move tempo.
- **Network (P2P)** — two players sharing one room code. Signalling and connection use [Trystero](https://github.com/dmotz/trystero) (nostr strategy, no backend of our own). The host runs the authoritative simulation; the guest sends moves and receives snapshots. Colors are random on the first game and alternate on rematches.

## Running locally

The P2P path (dynamic module import + WebRTC) needs `http(s)`, not `file://`. Serve from the project root:

```bash
python -m http.server 8123
# then open http://localhost:8123/
```

To play over the network, open two tabs (or share the link), pick the same code on the **Network** tab, and press **Connect** on both.

## PWA

Installable and offline-capable (single-player) via `manifest.webmanifest` and `service-worker.js`, which precaches the app shell. Icons and the OG image are generated by `scripts/make-icons.py` (Pillow) from the `favicon.svg` motif.

## Structure

```
index.html             markup, meta, PWA wiring
css/styles.css         styles (on top of niokit tokens)
js/state.js            constants and shared state
js/engine.js           rules, moves, arrival resolution
js/ai.js               AI (king defense, sniping, random moves)
js/net.js              P2P via Trystero
js/game.js             game start / start screen
js/render.js           board build + per-frame render
js/ui.js               input, controls, game loop, init
js/i18n.js             English / Russian strings
service-worker.js      offline app-shell cache
scripts/make-icons.py  icon + OG image generator
```

## Dependencies (via CDN)

- [niokit](https://github.com/anyalink99/niokit) — design tokens and UI components.
- [Trystero](https://github.com/dmotz/trystero) — P2P connection and signalling.
